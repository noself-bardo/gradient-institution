Object Name: History.

Purpose: History explains the institutional trace object in The Gradient. It records changes, reviews, steward actions, trust shifts, revisions, and status transitions across institutional objects without turning those changes into canon or authority by default.

Constitutional Role: History protects The Gradient from memory loss and false continuity. It makes institutional change inspectable, accountable, reversible when appropriate, and available for later review. It preserves the distinction between a record of change and a judgment of authority. It allows stewards to see how an object became what it currently is without treating the path as final truth.

Boundaries: History may attach to Sources, Evidence, Cards, Claims, Relationships, Engines, Decisions, and other institutional objects. It follows institutional action. It is not Archive. It is not Canon. It is not Validation. It is not Promotion. It is not Source Review. It is not a database audit log. It is not a software changelog. It is not a timeline infographic. It is not final authority. It does not prove that a change was correct. It records what changed, why it changed, who or what changed it, what state shifted, and what remains reviewable.
