## Purpose

Automation Boundary defines which tasks automation may assist and which actions it may never authorize.

## Constitutional Role

It matters because automation must not ratify, admit, approve, canonize, or replace human stewardship.

## Boundaries

Automation Boundary is distinct from Engine because an engine transforms inputs, while this boundary limits automated authority. It is distinct from Product / Platform Boundary because it focuses on automation's forbidden authority claims.

## Inputs / Outputs

### Inputs

- automation proposal
- allowed support task
- forbidden action
- human review need
- authority limit

### Outputs

- allowed assistance
- blocked action
- review route
- boundary note
- human-decision reminder

### Non-Outputs

- admission
- approval
- ratification
- canonization
- final authority

## Required Labels

- Automation Boundary
- Allowed Assistance
- Blocked Action
- Human Review
- No automated canon
- Automation cannot approve

## Forbidden Concepts

- canon by default
- visual approval as authority
- automatic promotion or ratification
- software interface or dashboard
- production workflow
- prompt engineering
- QC process as subject matter
- image gate as subject matter
- conversation summary
- mystical seal or certification badge
- object collapse into adjacent roadmap plates
