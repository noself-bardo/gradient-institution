## Purpose

Status State records where material currently sits in the institutional status vocabulary.

## Constitutional Role

It matters because explicit status prevents sandbox material from silently appearing admitted, validated, promoted, or canonical.

## Boundaries

Status State is distinct from Trust State because it records institutional position rather than confidence. It is distinct from Reclassification because it shows current state, while reclassification changes it.

## Inputs / Outputs

### Inputs

- object
- current status
- status reason
- status trace
- allowed next review

### Outputs

- status label
- state trace
- allowed next step
- boundary note
- review need

### Non-Outputs

- canon
- approval by label
- automatic promotion
- validation proof
- truth

## Required Labels

- Status State
- Sandbox

- Validated

## Forbidden Concepts

- canon by default
- visual approval as authority
- automatic promotion or ratification
- software interface or dashboard
- production workflow
- prompt engineering
- QC process as subject matter
- image gate as subject matter
- conversation summary
- mystical seal or certification badge
- object collapse into adjacent roadmap plates

## Diagram Family

State Plate.

Structure: A state band with named statuses and a highlighted current state, plus allowed next review without automatic movement.

## Visual Center

A state band with named statuses and a highlighted current state, plus allowed next review without automatic movement.

## Orientation

Landscape or near-square is acceptable when it improves legibility of the named structure.

## Negative Prompt

No progress bar to canon, no maturity model, no achievement badge, no traffic-light dashboard, no promotion ladder, no final seal. No paper mockup, no shadows, no perspective, no UI screen, no dashboard, no app card, no corporate infographic, no colorful categories, no neon, no random glyphs, no prompt engineering, no QC document, no image-generation process, no image gate, no conversation summary, no automatic canon language, no final authority language, no invented metadata.

## Compiler Notes

Render only Status State. Keep adjacent objects subordinate or absent unless listed as labels. Use GFVP field atlas grammar with matte black background, metallic silver and pale institutional gray typography and linework only, strict modular grid, high negative space, thin functional rules, and Sandbox / Candidate status language.
