## Purpose

Reentry Protocol restores context, boundaries, status, and resume point before work begins again.

## Constitutional Role

It matters because interrupted work should not restart from memory collapse or conversational drift.

## Boundaries

Reentry Protocol is distinct from Session Closeout because it begins the next session rather than ending the prior one. It is distinct from Archive because it uses preserved traces without being the preservation container.

## Inputs / Outputs

### Outputs
