## Purpose

Archive preserves institutional memory by holding source-bearing material, status traces, and retrieval paths without converting preservation into approval, canon, or visual lock.

## Constitutional Role

Archive matters because The Gradient needs durable memory that can be found again while remaining visibly separate from History, narrative shaping, public access, and approved visual output.

## Boundaries

Archive is not History: History records changes over time, while Archive stores preserved material and its retrieval path. Archive is not Book of Becoming: Book of Becoming shapes narrative memory, while Archive preserves without narration. Archive is not Source: Source is origin/provenance, while Archive is custody of retained material. Archive is not Commons: Commons is shared access, while Archive can preserve material before or beyond common use. Archive is not Canon, and it is not Approved Outputs; preservation does not ratify, approve, validate, promote, or visually lock anything.

## Inputs / Outputs

### Inputs

- source-bearing material
- origin or provenance trace
- current status label
- preservation reason
- retrieval requirement
- access limit

### Outputs

- preserved archive item
- archive custody note
- retrieval path
- status-preserved marker
- non-canon custody reminder
- access boundary note

### Non-Outputs

- history trace
- becoming narrative
- common-access publication
- canon entry
- approved output
- source authority

## Required Labels

- generic roadmap-object language

- software interface or dashboard
- public marketing or social-media treatment
- production workflow as subject matter
- prompt engineering or QC process as subject matter
- image gate as subject matter
- mystical seal or certification badge
- collapse into adjacent Pack B or governance objects

## QC Requirements

Pass only if:

- the central form is a custody vessel, not a library scene or software archive
- History, Book of Becoming, Source, Commons, Canon, and Approved Outputs remain visibly distinct
- preservation is shown as custody and retrieval, not approval
- labels include Not Canon and Not Approved Output
- retrieval paths do not cross into canon or approval lanes
- inputs, outputs, non-outputs, and labels match this specification
- no forbidden generic substrate language appears
