# GFVP Orientation Doctrine - Gate Boundary Plates

Status: Locked Sandbox Production Doctrine
Date: 2026-07-03
Originating Plate: P002 - Admission Gate

## Decision

Gate / Boundary Plates may use landscape orientation.

Landscape orientation is approved when the institutional object is best understood as a lateral crossing from input, through boundary, into next-action status.

## Rationale

The Admission Gate is not a portrait object, organ map, icon, or symbolic frontispiece.

It is a threshold object.

The diagram logic is:

- input approaches
- gate examines
- next-action exits branch

That structure benefits from horizontal space.

## Orientation Classes

| Orientation | Best Fit | Notes |
|---|---|---|
| Portrait | Identity plates, organ maps, constitutional frontispieces, symbolic structures | Use when one central object should read monumentally from top to bottom. |
| Landscape | Gates, boundaries, crossings, decision exits, process anatomy | Use when institutional logic moves across a threshold. |
| Square or Near-Square | Matrices, registries, capability maps | Use when the object requires balanced axes or central comparison. |

## Locked Constraint

Orientation must follow object logic.

Orientation must not be chosen for novelty, visual variety, or convenience alone.

## Applied Lock

P002 - Admission Gate is approved as a landscape Gate / Boundary Plate.

This does not approve every future landscape plate by default.

Future landscape plates must declare their object class and diagram reason.
