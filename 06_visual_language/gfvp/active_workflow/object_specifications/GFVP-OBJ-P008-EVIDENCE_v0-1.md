## Purpose

Evidence explains the bounded support object in The Gradient.

It is material used to ground Cards, Claims, and Relationships without making them automatically true, validated, canonical, or promoted.

## Constitutional Role

Evidence protects institutional reasoning from unsupported assertion.

It makes support inspectable, source-linked, scoped, and reviewable.

It preserves the distinction between support and proof.

It allows stewards to see why a Card, Claim, or Relationship is grounded without treating that grounding as final truth.

## Boundaries

Evidence follows Admission Gate, Source Review, and Metabolization Boundary.

It is a derived institutional support object.

It is not raw source material.

It is not the same object as Source.

It is not a Card.

It is not a Claim.

It is not a Relationship.

It is not Canon.

It is not Validation.

It is not Promotion.

It is not final proof.

It does not make a Claim true by existing.

It records support, scope, source linkage, trust state, and review history.

## Diagram Family

Support / Grounding Plate.

Primary structure: one central Evidence object grounding a subordinate supported object.

Secondary structure: small tethers to Source Link, Counterevidence, Limits, and Review History.

Do not use a legal proof board.

Do not use a full citation network.

Do not use a database schema.

Do not use a file or folder system.

Do not use an app interface.

## Visual Center

One severe central object labeled `EVIDENCE`.

The Evidence object contains the required evidence fields as internal facets, bands, or rows.

A subordinate anchor labeled `SUPPORTED OBJECT` may appear below or beside the Evidence object.

Small external tethers connect the Evidence object to:

- Source Link
- Counterevidence
- Limits
- Review History

The visual hierarchy must make Evidence more important than the supported object and tethers.

The plate must communicate bounded support, not final proof, verdict, validation, or canon.

## Orientation

Near-square or portrait is preferred.

Landscape is acceptable only if grounding tethers become unreadable.

The orientation reason is support-object anatomy and inspection, not threshold crossing, ledger readability, or lateral connection.
