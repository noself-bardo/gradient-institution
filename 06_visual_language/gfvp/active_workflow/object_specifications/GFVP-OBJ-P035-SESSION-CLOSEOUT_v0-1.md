## Purpose

Session Closeout records what changed, what locked, what remains living, and what must be resumed later.

## Constitutional Role

It matters because work sessions need clean endings that preserve state without pretending unfinished material is complete.

## Boundaries

Session Closeout is distinct from Confirmation Receipt because it summarizes session state, not one explicit decision. It is distinct from Archive because it routes what should be preserved but is not itself the archive.

## Inputs / Outputs

### Inputs

- session work
- changed material
- lock candidate
- open question
- next handoff

### Outputs

- closeout note
- locked list
- living list
- next action
- reentry cue

### Non-Outputs

- approval
- canon
- final publication
- image generation
- archive completion

## Required Labels

- Session Closeout
- Changed Material
- Locked Items
- Living Items
- Next Action
- Closeout is not approval
