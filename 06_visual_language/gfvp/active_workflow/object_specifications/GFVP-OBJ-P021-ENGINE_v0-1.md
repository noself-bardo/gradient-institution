## Purpose

Engine is a repeatable transformation object that takes named inputs through an operating rule to produce bounded outputs.

## Constitutional Role

It matters because transformation needs explicit limits, traces, and failure signals so an engine cannot become hidden authority.

## Boundaries

Engine is distinct from Practice Framework because it transforms material through rules, while a practice guides human attention. It is distinct from Automation Boundary because it names a transformation object, not the limits on automation authority.

## Inputs / Outputs

### Inputs

- defined input
- operating rule
- constraint
- run context
- failure condition

### Outputs

- defined output
- operation trace
- failure signal
- registry status
- review need

### Non-Outputs

- approval
- canon
- autonomous authority
- human replacement
- unbounded transformation
