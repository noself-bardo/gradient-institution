## Purpose

Failure Mode names a repeatable breakdown pattern with trigger, symptom, consequence, and prevention note.

## Constitutional Role

It matters because failure should become institutional memory and prevention, not blame or hidden repetition.

## Boundaries

Failure Mode is distinct from Capability Gap because it records breakdown in action, not merely absence. It is distinct from Output QC because it names a reusable pattern, not one candidate review.

## Inputs / Outputs

### Inputs

- failure event
- trigger condition
- observed symptom
- consequence
- correction note

### Outputs

- failure pattern
- prevention note
- archive trace
- review trigger
- avoidance cue

### Non-Outputs

- blame
- punishment
- canon
- approval
- single-output verdict

## Required Labels

- Failure Mode
- Trigger Condition
- Observed Symptom
- Prevention Note
- Archive Trace
- Failure becomes memory

## Forbidden Concepts

- canon by default
- visual approval as authority
- automatic promotion or ratification
- software interface or dashboard
- production workflow
- prompt engineering
- QC process as subject matter
- image gate as subject matter
- conversation summary
- mystical seal or certification badge
- object collapse into adjacent roadmap plates

## Orientation

Landscape or near-square is acceptable when it improves legibility of the named structure.

## Negative Prompt

No hazard sign poster, no blame board, no shame label, no disaster illustration, no bug tracker UI, no red failure stamp. No paper mockup, no shadows, no perspective, no UI screen, no dashboard, no app card, no corporate infographic, no colorful categories, no neon, no random glyphs, no prompt engineering, no QC document, no image-generation process, no image gate, no conversation summary, no automatic canon language, no final authority language, no invented metadata.

## Compiler Notes

Render only Failure Mode. Keep adjacent objects subordinate or absent unless listed as labels. Use GFVP field atlas grammar with matte black background, metallic silver and pale institutional gray typography and linework only, strict modular grid, high negative space, thin functional rules, and Sandbox / Candidate status language.
