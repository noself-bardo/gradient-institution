## Purpose

Source explains the institutional origin object in The Gradient.

It is the traceable material anchor from which Evidence, Cards, Claims, Relationships, and later interpretive objects may draw support.

## Constitutional Role

Source protects The Gradient from unsupported, contextless, or falsely certain knowledge.

It preserves memory of where material came from before the institution interprets, metabolizes, validates, promotes, or canonicalizes anything.

It keeps provenance visible even when provenance is incomplete.

It preserves the distinction between origin, support, interpretation, and authority.

## Boundaries

Source follows Admission Gate and Source Review.

It may precede or support Evidence.

It is not Evidence.

It is not a Card.

It is not a Claim.

It is not a Relationship.

It is not Source Review.

It is not Metabolization.

It is not Canon.

It is not Validation.

It is not Promotion.

It is not final proof.

It does not become authoritative because it is preserved.

It records origin, provenance, source type, trust limits, and institutional handling status.

## Visual Center

One severe central object labeled `SOURCE`.

The Source object contains the required source fields as internal facets, bands, or rows.

Small external tethers connect the Source object to:

- Related Evidence
- Related Cards
- Related Claims
- Review History

A provenance marker must remain visible.

The visual hierarchy must make Source more important than its downstream tethers.

The plate must communicate origin and provenance, not proof, validation, canon, or storage.
