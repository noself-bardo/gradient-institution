## Purpose

Capability Gap marks a needed capacity that is absent, weak, unsupported, or not yet preservable.

## Constitutional Role

It matters because explicit absence prevents the institution from pretending it can do what it cannot yet do.

## Boundaries

Capability Gap is distinct from Failure Mode because a gap may exist before failure. It is distinct from Uncertainty / Absence because it concerns capacity, not evidence or knowledge.

## Inputs / Outputs

### Inputs

- needed capability
- observed absence
- support missing
- risk context
- evidence note

### Outputs

- gap statement
- risk note
- support need
- review priority
- possible path

### Non-Outputs

- capability creation
- blame
- failure verdict
- approval
- canon

## Required Labels

- Capability Gap
- Needed Capability
- Observed Absence
- Support Missing
- Risk Note

## Forbidden Concepts

- canon by default
- visual approval as authority
- automatic promotion or ratification
- software interface or dashboard
- production workflow
- prompt engineering
- QC process as subject matter
- image gate as subject matter
- conversation summary
- mystical seal or certification badge
- object collapse into adjacent roadmap plates

## Diagram Family

Absence / Gap Plate.

Structure: A visible gap between needed capability and available support, with risk and review markers around the absence.

## Visual Center

A visible gap between needed capability and available support, with risk and review markers around the absence.

## Orientation

Landscape or near-square is acceptable when it improves legibility of the named structure.

No broken-machine metaphor, no blame chart, no backlog ticket, no deficit score, no red-alert dashboard, no failure stamp. No paper mockup, no shadows, no perspective, no UI screen, no dashboard, no app card, no corporate infographic, no colorful categories, no neon, no random glyphs, no prompt engineering, no QC document, no image-generation process, no image gate, no conversation summary, no automatic canon language, no final authority language, no invented metadata.
