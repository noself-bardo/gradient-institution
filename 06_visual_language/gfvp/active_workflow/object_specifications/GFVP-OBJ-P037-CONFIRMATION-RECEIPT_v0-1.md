## Purpose

Confirmation Receipt records a specific explicit confirmation, its actor, scope, status, and limits.

## Constitutional Role

It matters because The Gradient must know exactly what was approved, locked, rejected, deferred, or left untouched.

## Boundaries

Confirmation Receipt is distinct from Decision because it records explicit confirmation of an action or status. It is distinct from Steward Review because it records confirmation after review or instruction, not the review process itself.

## Inputs / Outputs

### Inputs

- requested action
- explicit response
- actor or steward
- scope note
- timestamp trace

### Outputs

- receipt record
- confirmed status
- limit note
- trace link
- non-covered item

### Non-Outputs

- unscoped approval
- canon
- automatic promotion
- image generation
- hidden consent
