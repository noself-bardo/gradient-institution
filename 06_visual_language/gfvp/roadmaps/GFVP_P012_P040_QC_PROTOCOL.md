# GFVP P012-P040 QC Protocol

Status: Ready for Codex Handoff
Date: 2026-07-04
Purpose: QC criteria for completing the remaining GFVP text-production layer.

## QC Principle

Every plate must remain one institutional object.

If an object begins to explain a whole workflow, chapter, ecology, or product system, it fails and must be narrowed.

## Object Specification QC

### Pass 01 - Source Authority

Pass if:

- the spec names appropriate Gradient source material
- the spec does not rely on conversation momentum as its authority
- the object belongs to the locked roadmap

Fail if:

- the spec invents institutional doctrine
- the spec treats visual style as source authority
- the spec imports software concepts as institutional primitives

### Pass 02 - Object Boundary

Pass if:

- the object is clearly defined
- the object names what it is not
- adjacent objects remain distinct
- the object has bounded inputs and outputs

Fail if:

- the object collapses into another roadmap object
- the object produces canon, validation, promotion, or authority outside its role
- the object becomes a workflow dashboard or product feature

### Pass 03 - Diagram Fit

Pass if:

- the diagram family matches the institutional function
- the visual center is singular
- subordinate components remain subordinate

Fail if:

- the diagram invites corporate infographic drift
- the diagram needs multiple centers
- the object wants to become a synthesis plate before its components exist

### Pass 04 - Required Labels

Pass if:

- labels are exact
- labels are short enough for image generation
- labels encode institutional boundaries

Fail if:

- labels are vague
- labels contain unqualified canon language
- labels invite image models to invent categories

### Pass 05 - Negative Prompt

Pass if:

- common drift modes are explicitly excluded
- production-process drift is excluded
- visual and conceptual exclusions are both present

Fail if:

- the negative prompt omits dashboards, UI, corporate infographics, mystical symbols, random glyphs, production workflow, prompt engineering, QC, image gate, conversation summary, or canonization drift

## Compiled Prompt QC

### Pass 01 - Compiler Discipline

Pass if:

- the prompt reads as a mechanical compilation of the object spec
- the prompt does not become a creative brief
- the prompt starts from the object, not the conversation

Fail if:

- the prompt asks the image model to interpret the institution broadly
- the prompt uses vibes as instructions
- the prompt references the current chat as source material

### Pass 02 - Object Fidelity

Pass if:

- one object is rendered
- non-outputs are blocked
- adjacent objects are not made dominant

Fail if:

- the prompt combines multiple roadmap objects
- the prompt lets outputs exceed the object's authority

### Pass 03 - Visual Grammar

Pass if:

- the established GFVP field atlas grammar is present
- metadata is exact
- status band is exact
- footer uses candidate language

Fail if:

- prompt permits color accents
- prompt permits corporate dashboard styling
- prompt permits canonical representation language

### Pass 04 - Label Hygiene

Pass if:

- required labels are exact
- title text is exact
- status text is exact
- metadata dates and IDs are exact

Fail if:

- labels contain typos
- status is misspelled
- catalog identifier is wrong
- object class and plate type contradict the roadmap

### Pass 05 - Image-Gate Readiness

Pass if:

- prompt can be used for one named plate
- prompt warns not to infer from surrounding conversation
- prompt blocks production-process drift

Fail if:

- prompt could generate a meta-document
- prompt could generate a batch process poster
- prompt could generate a software interface

## Output QC for Future Image Runs

When images are eventually generated, run output QC before approval:

- Object Fidelity
- Diagram Family
- Label Hygiene
- Visual Hierarchy
- Boundary Control
- Visual Grammar
- Drift Watchpoints
- Recommendation

Possible statuses:

- Pass / Ready for User Approval
- Visual Pass / Metadata Revision Required
- Metadata Pass / Label Revision Required
- Conceptual Fail / Regenerate
- Failure Mode / Archive

## Known Failure Modes to Watch

- Corporate Infographic Drift
- Meta-Conversation Drift
- Software Dashboard Drift
- Legal Authority Drift
- Mystical Canon Drift
- Timeline Drift
- Certification Badge Drift
- Overloaded Synthesis Plate
- Label Mutation
- Status Typo
- False Canonization

## Recommended Image-Gate Strategy

Do not generate P012-P040 as one uncontrolled image batch.

Recommended visual sequence:

1. Finish all text packets.
2. Generate P012-P015 one at a time.
3. Review and lock governance/authority visual grammar.
4. Generate Pack B in pairs.
5. Generate Packs C-D in small batches after prompt QC.
6. Generate Pack E carefully because status/failure plates are drift-prone.
7. Generate P040 only after P001-P039 are approved or intentionally waived.
