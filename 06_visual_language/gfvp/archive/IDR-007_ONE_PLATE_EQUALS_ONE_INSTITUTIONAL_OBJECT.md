# IDR-007 - One Plate Equals One Institutional Object

Status: Accepted / Sandbox Production Doctrine
Date: 2026-07-03
Related Package: Gradient Foundational Visual Package

## Decision

One plate equals one institutional object.

A field atlas plate should explain one bounded Gradient organ, gate, process state, relationship, or constitutional object.

## Context

P001 succeeded because it had one dominant institutional object.

Subsequent failed generations showed that broad plate prompts and conversational generation instructions invite model drift.

The model can substitute:

- corporate infographic structure
- production workflow
- QC conversation
- prompt engineering meta-language
- invented institutional language

## Rationale

Atomic object specifications preserve:

- source authority
- one diagram family
- one visual center
- one status boundary
- one institutional function
- one compiler target

## Consequences

GFVP should refactor from broad plate prompts into object specifications.

Images should be generated only from completed object specifications.

Synthesis plates may exist later, but only after component objects pass QC.

## Boundary

This is accepted Sandbox production doctrine.

It does not amend the Constitution.
