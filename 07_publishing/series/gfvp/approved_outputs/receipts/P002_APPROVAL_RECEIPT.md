## Approval

P002 is approved as a Gate / Boundary Plate format.

The landscape orientation is approved as intentional for this object class.

## Rationale

Admission Gate is a lateral crossing object:

- artifact approaches from the left
- gate occupies the central threshold
- possible next-action statuses exit to the right

Landscape supports the object logic because it gives the intake artifact, central gate, and status exits enough room to remain legible without turning the plate into a compressed lifecycle diagram.

## Locked Format Doctrine

Gate / Boundary Plates may use landscape orientation when the object depicts:

- entry across a threshold
- one central gate or boundary
- plural next-action exits
- source-side input and decision-side output

Landscape orientation is not a deviation from GFVP visual grammar for this plate class.

## Boundary

This approval applies to P002 and to the Gate / Boundary Plate format doctrine.

It does not approve all future landscape plates by default.

Future landscape use must be justified by object class and diagram logic.
