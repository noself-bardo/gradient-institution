## Rationale

P005 Card is not a Claim, Relationship, folder, database row, kanban card, dashboard, validation process, promotion process, or canonization process.

It is a human-facing knowledge object:

- one central Card
- internal card fields
- limited subordinate tethers
- source, claims, relationships, practice, and history as linked supports
- explicit non-folder and non-claim boundary
- exact metadata band

## Locked Format Doctrine

Object Anatomy Plates may use portrait or near-square orientation when the object requires a central inspectable form and internal field anatomy.

Portrait/near-square orientation for P005 is justified by human-facing object readability.

It is distinct from:

- P002 Gate / Boundary Plate landscape logic, justified by lateral threshold crossing
- P003 Ledger / Review Plate landscape logic, justified by ledger readability and disposition stamp spacing
- P004 Boundary / Conversion Plate landscape logic, justified by conversion from prior-state material to provisional outputs

## Boundary

This approval applies to P005 v0.1 and to the Card Object Anatomy Plate direction.

It does not approve all future object anatomy plates by default.

Future object anatomy plates must still pass object specification QC, compiled prompt QC, image generation, output QC, and explicit approval.
