# Platform Authority Matrix

| Platform | Owns | May execute | Must not become |
|---|---|---|---|
| The Gradient | Canon, governance, permissions, IDs, lifecycle | Institutional decisions | A vendor product |
| Notion | Specs, plans, dockets, decision records | Human control and documentation | Event bus or binary archive |
| Supabase | Structured state, APIs, audit events | Validated transactions | Canon authority |
| Google Drive | Durable source/export files | Asset retention and handoffs | Workflow engine |
| GitHub | Code, migrations, tests, deployment manifests | Reviewed implementation | Sensitive-data store |
| Cursor | Human-supervised edits | Local repository tasks | Source of truth |
| Codex | Bounded implementation | Tests, code, migrations, docs | Autonomous governor |
| Zo | Staging services, schedules, integration tests | Synthetic always-on workloads | Canonical archive |
| Local node | Restricted sources and sovereign runtime | Private processing | Untracked personal folder |
| ChatGPT | Analysis and orchestration | Packet preparation and review | Sole institutional memory |

## Core rule

A platform may execute Gradient logic, but no platform may redefine Gradient authority.
