# Two-Week Sprint Sequence

## Phase 1 — Baseline and boundaries
- Map all environments and sources of truth.
- Reconcile GitHub and Supabase.
- Define stable IDs and manifests.
- Lock privacy classification and create synthetic fixture.

## Phase 2 — Contracts
- Define packet JSON schema.
- Define lifecycle and illegal transitions.
- Create execution packet templates.
- Design additive Supabase thin-slice migration.

## Phase 3 — Operational layer
- Apply and test approved migration.
- Enforce RLS and credential boundaries.
- Build bounded APIs and immutable events.
- Define the human decision docket.

## Phase 4 — Zo staging
- Run read-only Zo inventory.
- Configure private synthetic-only staging.
- Deploy idempotent intake worker.
- Create health and synthetic-run automations.

## Phase 5 — Human surface and verification
- Build minimum docket interface.
- Run all four human dispositions.
- Run privacy, failure, retry, and rollback tests.
- Export and restore state.
- Document local replacement of Zo.
- Issue closeout receipt and Sprint 002 docket.
