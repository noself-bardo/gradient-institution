# 1. Ruling

CL-OPS-001 and CL-PROD-ARCH-001A are complementary. Neither supersedes the other.

- **CL-OPS-001 governs role execution and the required operating sequence.**
- **CL-PROD-ARCH-001A governs system structure, institutional boundaries, preservation rules, state separation, and the immediate lock set L-001 through L-016.**

Both remain subordinate to:

1. Crisis Liturgies Canon;
2. accepted Canon Change Reports;
3. the locked Crisis Liturgies Generation Standard;
4. a ratified release profile;
5. a frozen volume specification.

# 3. Conflict rule

When CL-OPS-001 and CL-PROD-ARCH-001A appear to conflict:

1. stop execution;
2. compare the exact provisions;
3. apply the higher authority hierarchy;
4. apply a named patch when one exists;
5. issue a conflict record if the provisions remain irreconcilable;
6. do not let an agent choose an interpretation.

# 4. Shared locked rulings

Both records affirm:

- Chachipti is bounded and subordinate;
- Codex and Cursor are execution systems, not institutional authorities;
- `CONCEPT_APPROVED` and `APPROVED_FOR_RENDER` govern deterministic builder execution;
- interactive ChatGPT image authorization is separate;
- frozen releases cannot be overwritten;
- final release decisions remain human;
- production must become specification-driven and reproducible.
