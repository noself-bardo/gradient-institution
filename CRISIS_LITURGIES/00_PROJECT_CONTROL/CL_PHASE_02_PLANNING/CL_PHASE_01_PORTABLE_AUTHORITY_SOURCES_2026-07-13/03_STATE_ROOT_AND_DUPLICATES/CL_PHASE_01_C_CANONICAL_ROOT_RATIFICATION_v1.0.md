# 2. Institutional split

The ratified architecture distinguishes:

- **active machine authority** from
- **frozen archival authority**.

The active repository may reference large binaries held in Drive through manifests and checksums.

The Drive archive may preserve copies of active outputs, but it may not generate or silently override active repository state.

# 3. Root-role rules

## Active repository rules

The active repository:

- is Git-governed;
- contains machine-readable specifications;
- contains builders, validators, schemas, and registries;
- records changes through commits;
- may not overwrite frozen releases;
- may not infer archive or publication state from file presence.

## Archive and mirror rules

The Drive archive:

- preserves frozen artifacts and source evidence;
- stores large binary releases and mirrors;
- retains historical and superseded material with explicit state labels;
- does not become the live production source merely because it contains the most files;
- does not authorize publication, archive admission, or active state.

# 4. Migration prohibition

This ratification selects the target architecture.

It does not authorize:

- moving existing files;
- renaming existing folders;
- deleting duplicates;
- consolidating mirrors;
- changing frozen releases;
- importing disputed collections;
- retiring any legacy root;
- running a builder.

All physical migration requires a separately approved Phase 02 migration plan and per-collection manifests.
