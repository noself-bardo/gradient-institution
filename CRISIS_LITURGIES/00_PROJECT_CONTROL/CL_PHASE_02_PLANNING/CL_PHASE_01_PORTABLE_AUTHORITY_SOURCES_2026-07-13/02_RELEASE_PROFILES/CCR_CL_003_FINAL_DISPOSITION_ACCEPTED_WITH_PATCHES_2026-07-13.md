# 1. Binding ruling

The Crisis Liturgies four-page architecture is governed by four invariant semantic functions:

1. `RELIC_ENTRY`
2. `WITNESS_RECORD`
3. `SYSTEM_TRANSLATION`
4. `ARCHIVE_DISPOSITION`

These functions are canonical semantic roles.

They are **not mandatory visible headings**.

Every new four-page Crisis Liturgies issue must declare a ratified release profile that maps its visible page labels to these four invariant functions.

# 2. Named patches

## CCR-CL-003-P01 — Semantic roles, not mandatory headings

The invariant functions govern meaning and sequence. A publication may use profile-specific display language rather than printing the machine-readable function names.

## CCR-CL-003-P02 — Preserve profile-specific display labels

Accepted profile mappings may use distinct reader-facing labels while retaining the same underlying semantic roles.

Initial admitted mappings:

| Invariant function | Archive Relic profile | Reader-Facing Liturgy profile | Expansion Volume profile |
|---|---|---|---|
| `RELIC_ENTRY` | Identity | Artifact | Encounter |
| `WITNESS_RECORD` | Primary Observation | Liturgical Hymn | Mechanism-Witness |
| `SYSTEM_TRANSLATION` | Failure Record | Gradient Translation | Institution-Expansion |
| `ARCHIVE_DISPOSITION` | Archive Closure | Archive Commentary | Residue-Turn |

## CCR-CL-003-P03 — No retroactive renaming

Existing frozen releases retain their original visible page labels, filenames, pagination, and internal language.

Retrospective metadata may map those pages to invariant functions without modifying the frozen artifact.

## CCR-CL-003-P04 — Mandatory release-profile declaration

Every new volume specification must declare:

- release profile ID;
- profile version;
- visible page labels;
- invariant-function mapping;
- page count;
- exceptions;
- ratification state.

A missing or unresolved mapping is a builder stop condition.

## CCR-CL-003-P05 — Special-volume exception

A Crisis Liturgies special volume may use more than four pages per issue.

A special-volume profile must:

- declare its own page sequence;
- preserve the four invariant semantic functions at the issue or movement level;
- identify any additional page functions;
- specify whether each invariant role appears once, recurs, or is distributed;
- receive separate human ratification.

The Death of the Bit is eligible for a special-volume profile. Its ten-page architecture is not forced into four visible pages.

## CCR-CL-003-P06 — Builder and compiler constraint

Chachipti and the deterministic builder may validate and compile declared mappings.

They may not:

- invent a missing profile;
- choose display labels;
- force a four-page layout onto a special volume;
- rename frozen releases;
- resolve an ambiguous mapping;
- infer profile membership from a folder name.

## CCR-CL-003-P07 — Visual palette remains profile-bound

Visual palette, typography, density, border logic, field color, and editorial treatment remain profile-specific unless separately ratified as universal law.

The semantic function code does not itself impose one visual style across all Crisis Liturgies volumes.

# 3. Authority relationship

This record is subordinate to Crisis Liturgies Canon and equal in authority to other accepted Crisis Liturgies Canon Change Reports.

It governs:

- semantic page-function architecture;
- release-profile mapping;
- profile declaration requirements;
- four-page and special-volume compatibility.

It does not authorize:

- file migration;
- image generation;
- rendering;
- archive admission;
- publication;
- deletion or deduplication.
