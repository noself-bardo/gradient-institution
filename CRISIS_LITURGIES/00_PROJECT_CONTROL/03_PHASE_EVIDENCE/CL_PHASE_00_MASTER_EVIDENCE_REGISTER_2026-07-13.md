# 2. Evidence classification

- **Authority Tier A:** locked canon, accepted change, or explicit human ratification.
- **Authority Tier B:** locked standard or operating protocol subordinate to canon.
- **Authority Tier C:** current control, QC, manifest, registry, or architecture record.
- **Authority Tier D:** handoff, runbook, build receipt, prompt package, or derived record.
- **Evidence Tier E:** chat or historical provenance that requires institutional capture.
- **Derived:** output existence without independent authority to define state.

Confidence states include VERIFIED, CORROBORATED, PARTIAL, UNVERIFIED, DISPUTED, and NEGATIVE FINDING.
